# Humanizer 繁體中文版

一個 Claude Code skill，專門移除繁體中文文字中的 AI 寫作痕跡，讓文章讀起來更自然、更像真人寫的。

## 安裝

### 推薦方式（直接 clone 到 Claude Code skills 目錄）

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/blader/humanizer-zhtw.git ~/.claude/skills/humanizer-zhtw
```

### 手動安裝／更新（僅 skill 檔案）

```bash
mkdir -p ~/.claude/skills/humanizer-zhtw
cp SKILL.md ~/.claude/skills/humanizer-zhtw/
```

## 使用方式

在 Claude Code 中呼叫 skill：

```
/humanizer-zhtw

[貼上你要處理的文字]
```

或直接請 Claude 去 AI 化：

```
請幫我把這段文字去 AI 化：[你的文字]
```

## 概述

英文版 `humanizer` 基於 [Wikipedia「AI 寫作特徵」](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing)指南。本繁體中文版在此基礎上，針對中文 AI 寫作的特有習慣延伸設計，特別是在科技商業語境中的高頻詞彙與固定句式。

### 核心原則

> AI 使用統計演算法猜測下一個應該出現的詞。中文 AI 的統計偏好集中在科技商業語境的高頻詞彙，避開這些詞，是讓文字回歸人味的第一步。

## 24 個偵測模式（含改前/改後範例）

### 內容模式

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 1 | **過度強調意義** | 「標誌著……演進史上的關鍵時刻」 | 「成立於 2010 年，負責整合各部會數位服務」 |
| 2 | **過度強調知名度** | 「曾被《天下》《商周》《聯合報》引用」 | 「她在 2024 年接受《天下》採訪時主張……」 |
| 3 | **表面化補充說明** | 「凸顯了……象徵著……彰顯了……」 | 移除或替換為具體來源 |
| 4 | **宣傳性語言** | 「坐落於令人嘆為觀止的花蓮」 | 「花蓮位於台灣東部」 |
| 5 | **模糊歸因** | 「專家認為扮演至關重要的角色」 | 「根據工研院 2023 年調查……」 |
| 6 | **公式化挑戰段落** | 「儘管面臨挑戰……仍持續蓬勃發展」 | 描述具體的問題和應對措施 |

### 語言模式

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 7 | **AI 詞彙** | 「賦能、生態系統、深度融合、協同效應」 | 說清楚實際在做什麼 |
| 8 | **迴避直接說「是」** | 「象徵著、扮演著……角色、作為……載體」 | 「是、有」 |
| 9 | **負面排比** | 「這不只是……更是……」 | 直接說重點 |
| 10 | **三連排** | 「洞見、靈感與人脈資源」 | 用自然的數量 |
| 11 | **同義詞輪換** | 「主角……核心人物……中心角色……英雄」 | 統一用「主角」 |
| 12 | **假範圍** | 「從遠古文明走到現代科技巔峰」 | 直接列出涵蓋主題 |

### 風格模式

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 13 | **破折號堆疊** | 「這項計畫——涵蓋……——將全面推動……——乃至……」 | 改用逗號或句號 |
| 14 | **過度粗體** | 「**賦能**、**生態鏈**、**差異化優勢**」 | 移除粗體 |
| 15 | **帶粗體標題的清單** | 「**使用者體驗：** 透過全新介面……」 | 改寫成完整段落 |
| 16 | **表情符號** | 「🚀 啟動階段：💡 核心洞察：」 | 移除表情符號 |

### 溝通模式

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 17 | **聊天殘留語** | 「希望以上內容對您有所幫助！歡迎隨時告知」 | 完全移除 |
| 18 | **知識截止免責聲明** | 「基於現有資訊，具體細節尚不明朗」 | 找到來源或直接刪除 |
| 19 | **奉承語氣** | 「好問題！您的觀察非常敏銳！」 | 直接回應內容 |

### 填充語與過度保守措辭

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 20 | **填充語** | 「為了達成這一目標」、「值得注意的是」 | 「為此」、直接說結論 |
| 21 | **過度保守** | 「或許可以認為……可能產生一定的潛在影響」 | 「這項政策可能影響結果」 |
| 22 | **泛泛正面結語** | 「前景一片光明，迎來更美好的明天」 | 具體的計畫或數字 |

### 中文特有模式

| # | 模式 | 改前 | 改後 |
|---|------|------|------|
| 23 | **企業 PPT 語言** | 「全面佈局產業鏈……精準觸達目標受眾……協同效應最大化」 | 說清楚實際在做什麼 |
| 24 | **「深度/賦能/重塑」三連發** | 「AI 技術正在深度賦能各行各業，重塑傳統商業模式」 | 給出具體的行業案例和數據 |

## 完整範例

**改前（AI 腔）：**
> 好問題！以下是相關整理，希望對您有所幫助！
>
> AI 輔助程式開發是大型語言模型變革潛力的持久明證，標誌著軟體開發演進史上的樞紐時刻。在當今快速演變的技術格局中，這些突破性工具深度賦能工程師的構思、迭代與交付全流程，凸顯了其在現代工作流程中不可或缺的角色。
>
> 核心價值主張清晰：精簡流程、強化協作、培育一致性。這不只是自動補全，更是大規模解放創造力的關鍵賦能工具。

**改後（去 AI 化）：**
> AI 程式輔助工具確實讓某些任務變快了。Google 2024 年的研究顯示，使用 Codex 的開發者完成簡單函數的速度比對照組快 55%，但在除錯或架構決策上沒有差異。
>
> 這些工具擅長處理樣板程式碼：設定檔、測試架構、重複的重構。它們不擅長的是知道自己什麼時候錯了。

## 與英文版的差異

| | `humanizer`（英文版） | `humanizer-zhtw`（繁體中文版） |
|---|---|---|
| 目標語言 | 英文 | 繁體中文 |
| 模式數量 | 24 個 | 24 個（含 2 個中文專屬） |
| 中文特有模式 | 無 | 企業 PPT 語言、「深度/賦能/重塑」三連發 |
| 引號規範 | 直引號 `"..."` | 書名號 `「」` |
